# אוטומציית Fireberry — ליד חדש נכנס לרצף החימום

מטרה: כשנכנס ליד חדש ל-Fireberry, הבוט מקבל אותו אוטומטית ומתחיל את רצף החימום בוואטסאפ.

## ההגדרה (במסך האוטומציות של Fireberry)

**טריגר:**
- אובייקט: **תלמיד** (Account)
- מתי: כשרשומה נוצרת/מתעדכנת והשדה **סטטוס** (statuscode) הופך ל-**"חדש"** או **"פנייה חוזרת"**

**פעולה: שליחת בקשת HTTP / Webhook**
- Method: `POST`
- URL: `https://cmtm-bot.onrender.com/lead`
- Headers:
  - `Content-Type: application/json`
  - `x-bot-secret: cmtm-wizup-7421`
- Body (JSON):
  ```json
  {
    "phone": "{telephone1}",
    "name": "{firstname}",
    "accountId": "{accountid}"
  }
  ```
  בתוך הסוגריים {} יש לבחור את שדות ה-Fireberry: טלפון ראשי, שם פרטי, מזהה התלמיד.

## מה קורה אז
1. הבוט יוצר/מאתר את הליד לפי הטלפון.
2. שולח את הודעת הפתיחה (welcome_intro_v2) מיד.
3. אם הליד לא עונה, שולח הודעת חימום כל 3 ימים (10 הודעות, 27 יום).
4. אם הליד עונה, הוא יוצא מהרצף והבוט החכם משתלט.
5. כל שיחה נשמרת על כרטיס התלמיד (אובייקט וואטסאפ 1012).

## כיבוי/הפעלה של החימום
מתג ראשי קיים: משתנה הסביבה `DRIP_ENABLED` (true/false). כרגע: true (פעיל).
לכיבוי זמני אפשר לעדכן מרחוק — פנה למפתח.

## בדיקה
שנה ידנית סטטוס של תלמיד בדיקה ל"חדש" → ודא שמתקבלת הודעת פתיחה בוואטסאפ למספר שלו.
